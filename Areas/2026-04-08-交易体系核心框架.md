# 交易体系核心框架

## 核心定位

> **我只做有基本面锚点的变化与修正。**

这句话具体展开，就是四个条件必须同时满足：

1. 基本面不是空的
2. 边际变化已经开始
3. 市场还没完全定价
4. 后续有验证路径

## 只做什么

| 类型 | 可以做的理由 |
|------|--------------|
| 成长预期差 | 有基本面支撑，也有价格弹性 |
| 困境 / 政策修复预期差 | 市场悲观过度，修复已出现 |
| 高质量龙头业绩验证 | 兑现更扎实，持有体验更稳定 |

## 明确不做什么

| 类型 | 不做的理由 |
|------|------------|
| 纯低估值慢修复 | 催化弱，时间成本高 |
| 高股息 / 类债配置 | 低弹性，不匹配真实偏好 |
| 高拥挤成长后半段 | 对波动、纪律和估值承压要求太高 |
| 纯题材 / 纯消息 / 纯情绪 | 缺少基本面锚点 |
| 无法验证的故事 | 只能靠想象支撑 |

## 开仓前五问

### 1. 我到底在挣什么钱？

- 如果连类型都说不清，这笔交易直接降级。

### 2. 未来 1 到 4 个季度，什么变量会变？

- 订单
- 价格
- 利润率
- 渠道
- 现金流
- 市场份额

### 3. 这个变量怎么验证？

- 财报
- 月度数据
- 订单公告
- 价格数据
- 渠道反馈
- 公司经营指标

### 4. 市场现在定价到哪一步了？

| 阶段 | 结论 |
|------|------|
| 还没看到 | 可以重点看 |
| 刚开始看到 | 可以做 |
| 已形成一致预期 | 谨慎做 |
| 已经过度拥挤 | 不做 |

### 5. 这笔交易是 A / B / C 哪一类？

| 等级 | 定义 | 仓位 |
|------|------|------|
| A | 主路径，逻辑清楚，验证扎实，市场未充分定价 | 10% - 25% |
| B | 逻辑尚可，但验证不完整或价格一般 | 5% - 10% |
| C | 题材、消息、讲不清的 | 0% 或极小仓位 |

## 仓位规则

### 允许加仓的前提

- 验证更强了
- 赔率仍然合理
- 市场还没完全透支

### 禁止加仓的情形

- 只是价格涨了
- 没有新增验证
- 估值已经透支

> **上涨先减少赔率，不会自动增加赔率。**

## 卖出语言必须统一

### 卖出只能来自三种触发

1. **逻辑失效**
   - 核心变量没变，或者反向变化。

2. **节奏走坏**
   - 逻辑还在，但价格结构明显走坏。

3. **兑现完成**
   - 市场已经充分定价，继续持有不划算。

### 卖出后允许买回

只要三个条件重新满足，就不必纠结“卖飞”：

- 逻辑继续强化
- 价格重新转强
- 节奏重新合适

## 当前最核心的系统问题

| 问题 | 表现 |
|------|------|
| 仓位和交易质量不匹配 | 差样本重仓，好样本轻仓 |
| 决策语言切换 | 买入、持有、卖出各用一套逻辑 |
| 把“能讲清逻辑”误当成“适合重仓” | 典型如格力、长江电力 |
| 容易被热度和比较效应带偏 | 成功样本里也会被别的票干扰 |

## 真正要练的四件事

1. 每笔交易先定义赚钱类型
2. 每笔交易只抓一个主变量
3. 仓位只跟机会质量挂钩
4. 只赚属于自己的钱

## 一句话版本

> **以后只做“有基本面锚点、有边际变化、有验证路径、市场未充分定价”的机会；其他机会先不碰。**
