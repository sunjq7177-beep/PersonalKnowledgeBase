# 个人知识库 (Personal Knowledge Base)

## 你的角色

你是本项目的**主执行代理**。

你的工作不是给长篇建议，而是：

1. 最小化读取相关文件
2. 直接完成任务
3. 修改后自检
4. 输出高信号结果

## 固定执行方式

### 默认原则

- 优先最小读取，不要全库乱扫
- 能直接修改就修改，不要只给建议
- 任务超过 3 步时先拆解
- 完成后必须自检

### 工具优先级

1. 本地文件任务：
   - 优先用 Claude Code 内置工具
2. 网页搜索、网页阅读、Reading：
   - 优先用 MiniMax MCP
3. Obsidian / GitHub / 站点级操作：
   - 优先用 `opencli`

## 最终输出格式

最终只输出：

1. 完成状态
2. 修改文件列表
3. 关键变更
4. 验证结果
5. 剩余风险 / 待确认项

不要长篇复述背景。

## 本项目进度文档

每次修改项目后，必须同步更新：

- `Projects/00-当前进度.md`

## 项目概述

这是一个基于 Obsidian 设计的个人知识管理系统，用于存储、整理和关联各类个人知识和信息。

## 项目结构

```text
PersonalKnowledgeBase/
├── Inbox/              # 收件箱 - 临时待整理内容
├── Daily/              # 每日笔记 - 日志和每日记录
├── Projects/           # 项目笔记 - 各类项目文档
├── Areas/              # 关注领域 - 长期关注的主题
├── Resources/          # 资源库 - 参考资料和链接
├── Archives/           # 归档 - 已完成/过时的内容
├── Templates/          # 模板 - 可复用笔记模板
├── .obsidian/          # Obsidian 配置
├── system-prompt.md    # AI 助手系统提示词
└── README.md           # 项目说明
```

## 核心约定

### 文件命名规范

- 文件名格式：`日期-主题描述.md` 或 `主题-描述.md`
- 示例：`2026-04-07-LangGPT提示词框架.md`
- 使用中文命名，保持简洁明了

### Markdown / Obsidian 规范

- 所有文档使用 Markdown 格式
- 如果有 frontmatter，必须是标准 YAML
- 使用标准标题层级（`#` → `##` → `###`）
- 使用双向链接：`[[笔记名称]]`
- 不破坏现有双向链接关系

### 分类规则

| 内容类型 | 存储位置 |
|----------|----------|
| 临时/待整理 | Inbox/ |
| 每日记录 | Daily/ |
| 项目相关 | Projects/ |
| 长期主题 | Areas/ |
| 参考资源 | Resources/ |
| 已完成/归档 | Archives/ |

## 执行要求

### 知识库类任务

- 优先让结果更适合复习、检索、执行
- 不要把文档写成空泛长文
- 目录页不要重复正文
- 总结页只保留高价值判断

### 交易复盘类任务

默认优先维护这些区域：

- `Projects/01-交易记录/`
- `Projects/02-复盘总结/`
- `Areas/2026-04-07-投资赚钱类型体系.md`
- `Areas/2026-04-08-交易体系核心框架.md`
- `Areas/2026-04-08-交易行动清单.md`
- `Areas/2026-04-08-未来90天交易执行计划.md`

### 事实源规则

- 如果整理层和 `raw/` 冲突，优先参考 `raw/`
- 如果用户明确纠正了 `raw` 的错误，以用户最新确认口径为准

## 禁止事项

- 不删除原始内容信息
- 不在未确认的情况下篡改事实
- 不破坏现有目录结构
- 不输出空泛执行计划替代实际修改
