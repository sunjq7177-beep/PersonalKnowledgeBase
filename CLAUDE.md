# 个人知识库 (Personal Knowledge Base)

## 项目概述

这是一个基于 Obsidian 设计的个人知识管理系统，用于存储、整理和关联各类个人知识和信息。

## 项目结构

```
PersonalKnowledgeBase/
├── Inbox/              # 收件箱 - 临时待整理内容
├── Daily/              # 每日笔记 - 日志和每日记录
├── Projects/           # 项目笔记 - 各类项目文档
├── Areas/              # 关注领域 - 长期关注的主题
├── Resources/          # 资源库 - 参考资料和链接
├── Archives/           # 归档 - 已完成/过时的内容
├── Templates/          # 模板 - 可复用笔记模板
├── .obsidian/          # Obsidian 配置
├── system-prompt.md    # AI 助手系统提示词
└── README.md           # 项目说明
```

## 核心约定

### 文件命名规范
- 文件名格式：`日期-主题描述.md` 或 `主题-描述.md`
- 示例：`2026-04-07-LangGPT提示词框架.md`
- 使用中文命名，保持简洁明了

### Markdown 格式规范
- 所有文档使用 Markdown 格式
- 必须包含 YAML frontmatter 元数据
- 使用标准标题层级（`#` → `##` → `###`）

### Obsidian 关联规范
- 使用双向链接：`[[笔记名称]]`
- 使用标签：`#主题 #标签`
- 主动建立知识节点关联

### 分类规则
| 内容类型 | 存储位置 |
|----------|----------|
| 临时/待整理 | Inbox/ |
| 每日记录 | Daily/ |
| 项目相关 | Projects/ |
| 长期主题 | Areas/ |
| 参考资源 | Resources/ |
| 已完成/归档 | Archives/ |

## 常用操作

### 存储新知识
接收用户发送的内容后：
1. 分析内容类型和主题
2. 选择合适的分类文件夹
3. 转换为 Markdown 格式，应用模板
4. 添加双向链接和标签
5. 写入文件并返回存储结果

### 搜索现有知识
- 搜索相关主题的现有笔记
- 建立新内容与现有知识的关联
- 更新相关笔记的链接

## 模板文件

- `Templates/Daily Note.md` - 每日笔记模板
- `Templates/Inbox Note.md` - 收件箱模板

## 禁止事项

1. 不删除原始内容信息
2. 不在回复中暴露完整文件路径
3. 不破坏现有双向链接关系
