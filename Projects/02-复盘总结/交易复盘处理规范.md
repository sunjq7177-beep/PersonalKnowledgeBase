# 交易复盘处理规范

## 这份规范解决什么

以后所有交易复盘，都按同一套结构整理，避免再次出现：

- 单笔复盘写得太长，复习时抓不到重点
- 目录页、总结页、单笔页互相重复
- 总结判断和 `raw/` 事实不一致

## 第一原则：`raw/` 是事实源

- 如果整理层和 `raw/` 冲突，以 `raw/` 为准。
- 时间、仓位、结果、关键逻辑不确定时，直接写“待确认”。
- 不允许为了把文字写顺而改动事实。

## 文档分层

### 1. 原始材料层

- `raw/`
- 只存原始输入、原文、报告、草稿。

### 2. 单笔交易层

- `Projects/01-交易记录/`
- 一笔交易一份文档。
- 作用：留事实、留结论、留可复制教训。

### 3. 总览层

- `Projects/02-复盘总结/交易复盘目录.md`
- 作用：导航、分类、时间线、阅读入口。

### 4. 判断层

- `Projects/02-复盘总结/交易复盘阶段性总结.md`
- 作用：沉淀当前最适合做什么、不适合做什么。

### 5. 规则层

- `Areas/2026-04-07-投资赚钱类型体系.md`
- `Areas/2026-04-08-交易体系核心框架.md`
- `Areas/2026-04-08-交易行动清单.md`
- `Areas/2026-04-08-未来90天交易执行计划.md`

## 单笔复盘固定结构

每篇单笔复盘必须按下面顺序：

1. 先看结论
2. 交易事实
3. 当时为什么买
4. 过程与关键转折
5. 复盘判断
   - 做对了什么
   - 做错了什么
   - 以后怎么处理同类机会

### 单笔复盘必须回答的五个问题

1. 这笔钱到底是什么钱
2. 结果如何
3. 它是正样本、反样本还是试错样本
4. 它适不适合作为未来主路径
5. 它对后续规则意味着什么

## 目录页固定内容

目录页只保留：

- 一页总览表
- 按样本性质归类
- 按赚钱类型归类
- 按时间线查看
- 推荐阅读顺序

目录页**不再重复**单笔复盘的完整正文。

## 阶段性总结固定内容

阶段性总结只保留高价值判断：

1. 当前最适合做什么
2. 当前最不适合做什么
3. 七笔样本分别说明了什么
4. 当前最核心的共性问题
5. 接下来 30 / 60 / 90 天怎么执行

## 新增一笔交易时，必须同步更新

### 必改文件

1. 对应单笔复盘
2. `交易复盘目录`
3. `交易复盘阶段性总结`
4. `Projects/00-当前进度.md`

### 视情况更新

- `投资赚钱类型体系`
- `交易体系核心框架`
- `交易行动清单`
- `未来90天交易执行计划`

当新增样本改变当前主判断时，上面四份规则层文档也必须更新。

## 标签和属性建议

单笔交易 frontmatter 至少包含：

- `created`
- `updated`
- `source`
- `tags`
- `money_type`
- `sample_type`
- `fit`
- `status`

## 风格要求

- 先结论，后细节
- 优先表格和短列表
- 一段话只说一件事
- 禁止空泛总结和重复解释
- 所有内容都要服务于“以后更容易复习和执行”

## 当前项目的固定判断

### 当前主路径

- 成长预期差
- 困境 / 政策修复预期差
- 高质量龙头业绩验证与重估

### 当前回避项

- 纯低估值慢修复
- 高股息类债配置
- 拥挤成长后半段重仓趋势
- 纯题材 / 纯消息 / 纯情绪接力
