# 个人知识库

这是一个以 Obsidian 为载体的个人知识库。当前最成熟、也最需要持续维护的部分，是基于 `raw/` 原始材料整理出来的交易复盘系统。

## 当前使用原则

- `raw/` 是原始材料区，只存原文、草稿、报告和外部资料。
- `raw/` 之外的内容，默认都属于整理层、索引层、总结层或执行层。
- 如果整理层和 `raw/` 冲突，以 `raw/` 为准，先修正事实，再改总结。
- 每次修改项目内容，都要同步更新 `Projects/00-当前进度.md`。

## 目录说明

| 目录 | 作用 | 当前重点 |
|------|------|----------|
| `raw/` | 原始材料，不直接当最终笔记用 | 交易原始复盘、阶段性总结、研究报告 |
| `Projects/01-交易记录/` | 单笔交易复盘 | 每笔交易一页，强调结论、事实、教训 |
| `Projects/02-复盘总结/` | 总览、阶段性结论、规范 | 用于复习和统一判断口径 |
| `Areas/` | 长期有效的方法、体系、执行规则 | 投资赚钱类型、交易框架、行动清单 |
| `Templates/` | 固定模板 | 日记、Inbox、交易复盘、交易计划卡片 |
| `Inbox/` | 临时待整理 | 暂存输入 |
| `Daily/` | 每日笔记 | 日常记录 |
| `Resources/` | 参考资料 | 外部资源归档 |
| `Archives/` | 已归档内容 | 历史材料 |

## 交易区推荐阅读顺序

如果是第一次进入交易区，建议按下面顺序看：

1. `Projects/02-复盘总结/交易复盘阶段性总结.md`
2. `Projects/02-复盘总结/交易复盘目录.md`
3. `Areas/2026-04-08-交易体系核心框架.md`
4. `Areas/2026-04-08-未来90天交易执行计划.md`
5. `Projects/01-交易记录/` 下的单笔复盘

## 交易相关文档分层

### 1. 理论层
- `Areas/2026-04-07-投资赚钱类型体系.md`
- 用来回答：到底在挣什么钱。

### 2. 规则层
- `Areas/2026-04-08-交易体系核心框架.md`
- `Areas/2026-04-08-交易行动清单.md`
- `Areas/2026-04-08-未来90天交易执行计划.md`
- 用来回答：以后只做什么、怎么做、怎么复盘。

### 3. 个案层
- `Projects/01-交易记录/*.md`
- 用来回答：某一笔交易具体发生了什么，属于什么样本。

### 4. 总览层
- `Projects/02-复盘总结/交易复盘目录.md`
- `Projects/02-复盘总结/交易复盘阶段性总结.md`
- 用来回答：所有个案放在一起后，哪些能复制，哪些不能。

## 维护规则

- 单笔交易复盘必须优先服务于“以后怎么复习”，不是服务于“把故事讲完整”。
- 阶段性总结必须只保留高频会回看的判断，不写大段重复解释。
- 目录页只做导航、对照、归类，不再重复单笔复盘正文。
- 新增交易时，至少同步更新：
  - 单笔复盘
  - 交易复盘目录
  - 交易复盘阶段性总结
  - 当前进度文档
